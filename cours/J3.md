# Module « Sous le capot » (J3-J4) — plan & objectifs

> **Version canonique du cours** : page Notion « Alegria › Cours Vibe coding 2026 › J3 J4 - Sous le capot » (toggles, callouts, exercices avec solutions). Ce fichier est l'**outline** côté repo.
>
> Module = **2 jours**, découpé en **chapitres** (pas de frontière J3/J4 rigide : on dose selon le groupe). Colonne vertébrale : **où tourne le code (serveur vs navigateur)**.

## Première moitié — Lire et naviguer (~5 h)

| # | Chapitre | Durée | Objectif | Geste actif |
|---|---|---|---|---|
| 1 | Mise en route : brancher VOTRE Supabase | ~60 min | cloner, migrer, seed, `.env.local`, comptes démo | exercice guidé + dépannage |
| 2 | Le concept-pivot : où tourne votre code ? | ~60 min | serveur vs client, frontière des secrets | prédire serveur/client (solution) |
| 3 | La carte du repo : l'arborescence EST le routeur | ~30 min | App Router, segment `[id]`, `layout` | prédire l'URL d'un fichier |
| 4 | Lire un Server Component (`page.tsx`) | ~45 min | `async/await`, requête serveur, props | modifier le tri/filtre |
| 5 | L'état vit dans l'URL (filtres, jour) | ~40 min | search params vs `useState` | partager un lien filtré |
| 6 | TypeScript en lecture | ~30 min | unions, `type Court`, `\| null` | survol/autocomplétion VS Code |
| 7 | Tailwind + shadcn | ~30 min | utilitaires CSS, composants dans le repo | casser/réparer une couleur |

## Seconde moitié — Ouvrir, modifier, juger (~6,5 h)

| # | Chapitre | Durée | Objectif | Geste actif |
|---|---|---|---|---|
| 8 | Le flux d'écriture : une Server Action | ~60 min | `'use server'`, chemin clic→base, porte d'auth | prédire les `revalidatePath` |
| 9 | Exercice fil rouge : un champ de bout en bout | ~75 min | 3 couches base→type→UI | feature complète (indices + solution) |
| 10 | Auth, session & route protégée | ~45 min | `getUser`, `redirect`, `proxy.ts` | accéder à une page protégée déconnecté |
| 11 | Lire un CRUD complet (admin) | ~40 min | create/read/update/delete | repérer le masquage du lien admin |
| 12 | Atelier « Juger l'IA » (4 cas) | ~90 min | anti-patterns serveur/client/secret | analyser 4 extraits (solutions) |
| 13 | Synthèse & pont vers J5-J6 | ~30 min | grille de lecture, failles à venir | récap |

**Total ≈ 11 h 35 de contenu** + pauses/déjeuner ⇒ ~2 jours pleins.

## Matériel
- `cours/exemples-juger-ia/` : les 4 cas (versions ❌/✅) du chapitre 12.
- `cours/notes-formateur.md` : minutage détaillé, pièges, démos live, emplacement des failles volontaires.
