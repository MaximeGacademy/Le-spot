# Atelier « Juger l'IA » — guide d'animation (chapitre 12)

> Le plan complet du module est dans `cours/J3.md` et, en version animable (toggles/solutions), dans Notion. Ce fichier détaille **comment faire tourner l'atelier de jugement** — le cœur de l'objectif « juger le code d'une IA ». Code support : `cours/exemples-juger-ia/`.

## Principe

Pour chaque cas : (1) on **montre la version ❌** au tableau, (2) on pose la question *« où tourne ce code, et qu'est-ce qui cloche ? »*, (3) on laisse 3-4 min de recherche en binômes, (4) on **révèle la ✅** et on nomme la règle. Toujours ramener à la colonne : *où tourne le code*.

## Les 4 cas

| Cas | Fichier | Anti-pattern | Règle à faire émerger |
|---|---|---|---|
| 1 | `cas-1-data-fetching.tsx` | `useEffect` + `fetch` côté client pour de l'affichage | lire les données dans un Server Component |
| 2 | `cas-2-secret-expose.tsx` | clé `service_role` côté client / préfixe `NEXT_PUBLIC_` sur un secret | seul ce qui est public porte `NEXT_PUBLIC_` |
| 3 | `cas-3-tout-en-client.tsx` | page entière en `'use client'` | serveur par défaut, client en îlots |
| 4 | `cas-4-action-non-gardee.ts` | Server Action qui croit un champ du client | revérifier identité + droits côté serveur |

## Pont avec l'app

- Le cas 1 contraste avec `src/app/page.tsx` (bonne pratique réelle).
- Le cas 3 contraste avec `day-view.tsx` (serveur) + `slot-button.tsx` (îlot client).
- Le cas 4 EST la faille volontaire de `src/app/actions/courts.ts` → on l'observe ici, on la corrige en J5-J6.

## Geste de clôture

Faire lister par les étudiants **tous les `'use client'` du repo** et justifier chacun. Objectif : ils repartent avec le réflexe « où tourne ce code ? » automatisé.
